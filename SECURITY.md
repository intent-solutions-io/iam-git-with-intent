# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 0.5.x   | Yes (Current) |
| 0.4.x   | Security fixes only |
| < 0.4   | No |

## Reporting a Vulnerability

**Do NOT file public GitHub issues for security vulnerabilities.**

### How to Report

- Email: security@intentsolutions.io
- Expected response: 48 hours acknowledgment, 7 days initial assessment

### What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (optional)

## Scope

Security reports are accepted for:

- CLI tool (`gwi`)
- Cloud Run services (API, Gateway, Webhook, Worker)
- SDK and packages
- Infrastructure configurations
- GitHub Actions workflows

## Safe Harbor

We will not pursue legal action against security researchers who:

- Act in good faith
- Avoid privacy violations and data destruction
- Give us reasonable time to respond before disclosure
